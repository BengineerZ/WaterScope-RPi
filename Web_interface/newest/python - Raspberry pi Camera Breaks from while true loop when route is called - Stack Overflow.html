    Raspberry pi Camera Breaks from while true loop when route is called

    Ask Question
    up vote 0 down vote favorite

    I am very new to python, Flask and raspberry pi so an help would be great.

    I am trying to get a camera to take photos via the camera every 2 seconds then save the photo. I then have a route to download the photo ("/image")

    When I start the app I get a preview on the monitor connected to the pi all works as I hope.

    when I connect to the server to see the photo via the url (/image) the preview ends on the monitor attached to the pi and the photo on the server is not updated.

    I don't see any errors and I can't see how it breaks out of the while True: loop

    I have the below code:

    from flask import Flask
from flask import send_file
from camera import camera

app = Flask(__name__)


@app.route("/")
def hello():
return "hello world"

@app.route("/image")
def get_image():
    filename = './images/max.jpg'
    return (send_file(filename, mimetype='image/jpeg'), 200)


if __name__ == "__main__" :
    app.run(host = '0.0.0.0',debug = True)
    camera = PiCamera()
    camera.resolution = (2592, 1944)
    while True:
        camera.start_preview()
        sleep(2)
        camera.capture('./images/max.jpg')
        camera.stop_preview()
